Gemini 3.5와 에이전트 실행 최적화

5 min read · May 21, 2026

1. AI/기술 트렌드

a. AI 모델 & 서비스

  • Gemini 3.5 Flash 출시: 복잡한 에이전틱 워크플로우 실행에 최적화된 모델로, 지능과 행동(Action)의 결합을 강조함.

→ 단순히 답변을 잘하는 모델보다 도구 호출(Tool-calling)과 다단계 추론 속도가 빠른 모델이 개발 환경의 주류가 될 것이다.

  • OpenAI 모델, 80년 된 기하학 난제 해결: '평면 단위 거리 문제'에 대한 추측을 AI 모델이 직접 증명하여 수학적 성과를 거둠.

→ AI가 단순한 코드 생성을 넘어 논리적 검증과 고차원 알고리즘 설계의 파트너로서 실질적인 효용을 증명하고 있다.

  • Nvidia, AI 에이전트 전용 CPU 시장 진출: 젠슨 황 CEO가 에이전트 구동에 최적화된 2,000억 달러 규모의 CPU 시장 가능성을 언급함.

→ 에이전트의 병렬 실행과 지연 시간 단축을 위한 하드웨어 계층의 변화가 시작되고 있다.

b. AI 기업 동향

  • Anthropic 수익성 개선: 2분기 매출이 100억 달러를 돌파하며 첫 분기 흑자 전환을 예고함.
  • OpenAI IPO 준비: 이르면 금요일 비밀리에 IPO 서류를 제출할 예정이라는 소식이 전해짐.

c. AI 연구 & 기술

  • DeepWeb-Bench: 에이전트의 웹 검색 및 증거 수집 능력을 평가하는 새로운 벤치마크 등장.

→ 에이전트가 얼마나 많은 소스에서 정보를 취합하고 장기적인 추론을 유지하는지가 향후 모델 성능의 핵심 지표가 될 것임.

  • Mem-π: 모델이 필요한 가이드를 온디맨드로 생성하는 적응형 메모리 프레임워크 제안.

→ 외부 메모리에서 정보를 검색하는 방식보다 에이전트가 상황에 맞는 지침을 스스로 생성하여 효율성을 높이는 방향으로 발전 중이다.

2. 개발 업데이트

a. Vibe Coding

  • Karpathy의 LLM 코딩 관찰 반영: LLM 코딩의 함정을 피하기 위한 'andrej-karpathy-skills' 프로젝트(CLAUDE.md)가 인기를 얻음.

→ 시니어 개발자의 경험을 에이전트의 규칙(System Prompt/Config)으로 이식하는 과정이 개발 생산성의 핵심이다.

  • Forge: Guardrail 레이어를 통해 8B 소형 모델의 에이전트 작업 성공률을 53%에서 99%로 향상.

→ 로컬 모델을 사용하더라도 단계적 강제(Step Enforcement)와 맥락 관리 레이어를 잘 구축하면 유료 대형 모델 수준의 신뢰성을 확보할 수 있다.

b. 개발 트렌드

  • Google I/O 2026 발표: Antigravity 2.0, WebMCP, Android 17 등 에이전트 네이티브 생태계를 위한 도구들이 대거 공개됨.

→ 웹과 모바일 환경 모두에서 AI 에이전트가 OS 레벨의 도구에 접근하는 방식이 표준화되고 있다.

  • GCC 16 업데이트: SARIF 형식 출력 지원 및 에러 메시지 가독성 개선.

→ 정적 분석 도구와 AI 에이전트가 컴파일 에러를 더 정교하게 파악하고 자동 수정할 수 있는 인프라가 강화됨.

c. 보안 & 공급망

  • GitHub VS Code 확장 프로그램 침해 사고: 직원 1명의 악성 확장 프로그램 설치로 3,800개 내부 저장소 노출 확인.

→ 확장 프로그램 마켓플레이스의 보안 검증이 완벽하지 않으므로, 개발팀 내 비공식 확장 프로그램 사용을 엄격히 제한해야 한다.

  • Next.js 16 'use cache' 디버거: 보이지 않는 캐시 동작을 모니터링하기 위한 커스텀 디버거 개발 사례 공유.

→ 프레임워크의 추상화 수준이 높아질수록 내부 동작을 시각화하는 커스텀 도구의 가치가 커진다.

3. AI Coding Assistant

  • Claude Code v2.1.146 업데이트: /simplify 명령어를 /code-review로 통합하고 노력 수준(high/low) 옵션을 추가함.

→ 에이전트에게 단순 수정을 요청하는 것을 넘어 아키텍처 수준의 깊은 리뷰를 단계별로 요구할 수 있게 됨.

  • Codegraph: Claude Code 및 Cursor를 위한 로컬 코드 지식 그래프 프로젝트 부상.

→ 토큰 사용량을 줄이면서도 프로젝트 전체 맥락을 에이전트에게 전달하기 위해 로컬 인덱싱 도구 활용이 필수적이다.

  • Claude Code 배경 세션 지원: /resume 명령어로 백그라운드에서 실행 중인 에이전트 작업을 이어받는 기능 강화.

→ 시간이 오래 걸리는 작업은 백그라운드 에이전트에게 맡기고, 개발자는 다른 태스크에 집중하는 멀티태스킹 환경이 강화됨.

  • MCP(Model Context Protocol) 도구 실패 이슈: 승인 UI 누락 등 MCP 도구 호출 관련 버그가 보고됨.

→ MCP 생태계가 확장됨에 따라 도구 실행 권한 관리와 안정성 확보가 개발자들의 주요 과제가 되고 있다.

5. 인사이트

Gemini 3.5 Flash와 Claude Code의 릴리스 노트는 공통적으로 '에이전틱 워크플로우'의 안정화에 집중하고 있다. 이제 개발자는 코드를 대신 써주는 AI를 넘어, 복잡한 프로젝트 구조를 이해하고 도구를 직접 실행하는 에이전트를 관리하는 '오케스트레이터'로 진화해야 한다. 특히 GitHub 확장 프로그램 침해 사례에서 보듯, 에이전트가 사용하는 도구와 확장 프로그램의 보안 신뢰성을 검증하는 프로세스를 개발 프로세스에 반드시 편입시켜야 한다.
