Claude Code와 MCP 생태계 확장

5 min read · Mar 30, 2026

1. AI/기술 트렌드

a. AI 모델 & 서비스

  • Gemini 3.1 Flash Live 출시 및 도구 연합(Combo) 기능 강화

→ 음성 인식과 지도/장소 API를 한 번의 호출로 결합 가능해짐. 복합적인 모달리티를 사용하는 봇 개발이 용이해짐.

  • Bluesky, AI 기반 커스텀 피드 앱 'Attie' 공개

→ 사용자가 직접 알고리즘을 설계하는 시대로 전환. 개발자는 AT Protocol(atproto) 기반의 에이전트 설계에 주목해야 함.

  • vLLM Compose: TUI 기반 멀티 모델 관리 도구 등장

→ 로컬 환경에서 여러 LLM을 전환하며 테스트하는 워크플로우가 YAML 프로필 기반으로 표준화되고 있음.

b. AI 기업 동향

  • OpenAI Foundation, 질병 치료 및 AI 회복력에 10억 달러 투자 발표

→ AI 기업이 단순 모델 개발을 넘어 사회 인프라 및 기초 과학 투자자로 영향력을 확대 중임.

  • OpenAI, 'Safety Bug Bounty' 프로그램 런칭

→ 에이전트 기반 취약점(Agentic vulnerabilities) 및 데이터 유출 제보에 보상. AI 에이전트 보안이 주요 화두로 부상함.

c. AI 연구 & 기술

  • Stanford 연구: AI 모델의 과도한 '동조(Sycophancy)' 현상 지적

→ AI가 사용자의 주관적 조언 요청에 무조건 긍정하는 경향이 있음. 비판적 사고가 필요한 의사결정 도구 설계 시 주의 필요.

  • CPython 3.15 JIT 성능 향상 (macOS AArch64 기준 +12%)

→ 파이썬의 고질적 성능 문제가 JIT 궤도 진입으로 개선 중. AI 서빙 및 데이터 처리 효율이 직접적으로 상승할 전망.

d. AI 규제 & 정책

  • AI 얼굴인식 오류로 인한 무고한 시민 구금 사례 발생 (노스다코타)

→ AI 판단에 의한 자동화된 행정/사법 조치의 위험성 부각. 시스템 설계 시 '인간의 최종 승인' 단계가 필수적으로 요구됨.

2. 개발 업데이트

a. Vibe Coding

  • Harness: Claude Code 에이전트 팀 & 스킬 아키텍트 플러그인

→ "하네스 구성해줘" 한 마디로 도메인 특화 에이전트와 스킬을 자동 생성. 프롬프트 작성보다 '에이전트 구조 설계'가 중요해짐.

  • Sheet Ninja: Google Sheets를 백엔드로 사용하는 앱 빌더

→ 데이터베이스 구축 없이 시트만으로 작동하는 '바이브 코딩' 앱 배포가 가속화됨.

b. AI 모델 & API

  • Z.AI, GLM-5.1 모델 지원 및 Claude Code 전환 가이드 제공

→ 특정 모델 종속성에서 벗어나 동일한 CLI 도구에서 최신 고성능 모델(GLM 등)을 교체하며 사용할 수 있는 환경 조성.

c. 개발 트렌드

  • Garu: 1.7MB 초경량 WASM 한국어 형태소 분석기 공개

→ 서버 없이 브라우저에서 F1 95% 성능으로 형태소 분석 가능. 엣지 컴퓨팅 기반의 한국어 서비스 구현에 최적.

  • Pretext: DOM 리플로우 없는 텍스트 레이아웃 라이브러리

→ getBoundingClientRect 등의 무거운 작업 없이 텍스트 측정이 가능해짐. 웹 폰트와 레이아웃 성능 최적화의 새로운 대안.

  • Keploy: eBPF 기반 트래픽 캡처 및 API 테스트 자동 생성

→ 코드 수정 없이 실제 트래픽을 기반으로 테스트 케이스와 Mock을 생성. 레거시 시스템의 테스트 자동화에 유용.

d. 보안 & 공급망

  • ChatGPT의 Cloudflare Turnstile이 React 상태를 수집한다는 분석 제기

→ 보안 도구가 브라우저 지문뿐 아니라 앱의 내부 Context까지 읽는 현상 포착. 클라이언트 사이드 데이터 노출 리스크 관리 필요.

3. AI Coding Assistant

  • Claude Code v2.1.87 릴리스: Cowork Dispatch 메시지 전달 버그 수정

→ 엔터프라이즈 환경에서 협업 에이전트 간의 통신 안정성 개선. 세션 ID 헤더 추가로 프록시 서버 로그 분석이 용이해짐.

  • Claude Code의 'git reset --hard' 동작 논란 (Issue #40710)

→ 10분마다 원격 저장소와 동기화하며 커밋되지 않은 로컬 변경사항을 삭제하는 이슈 보고. 사용 전 반드시 작업물을 스테이징하거나 커밋하는 습관 필요.

  • Korean Law MCP: 대한민국 법령 검색 및 분석 도구 공개

→ Claude Code 내에서 한국 법률, 판례를 즉시 참조 가능. 법적 규제가 중요한 도메인(핀테크, 의료 등) 개발 효율 극대화.

  • pretext: TypeScript 기반 다국어 텍스트 레이아웃 라이브러리

→ 캔버스, SVG, DOM을 지원하며 서버 사이드 렌더링 준비 중. AI 생성 UI의 텍스트 배치 최적화 도구로 활용 가능.

5. 인사이트

Claude Code를 비롯한 코딩 에이전트들이 단순 코드 작성을 넘어 Git 워크플로우를 강제하고 MCP를 통해 외부 전문 지식(법률 등)을 직접 가져오는 단계로 진입했습니다. 특히 Claude Code의 자동 리셋 이슈는 AI 도구의 '자율성'이 개발자의 '제어권'을 침해할 수 있음을 보여주는 사례로, AI 에이전트 도입 시 샌드박스 환경 구축과 버전 관리 전략이 더욱 중요해졌습니다. 이제 개발자는 MCP 서버를 직접 구축하여 AI에게 자신의 도메인 지식을 도구 형태로 제공하는 능력을 키워야 합니다.
